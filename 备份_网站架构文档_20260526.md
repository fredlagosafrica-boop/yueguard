## 四、技术架构说明

### 文件结构
```
yueguard/
├── index.html           # 主页面
├── script_new.js        # 主脚本（含版本号缓存控制）
├── ifa_content.js      # IFA流程指引（v2026052703）
├── wiki_content.js     # 百科全书
├── sales_content.js    # 进阶销售
├── referral_content.js # 港险渠道业务
├── materials_content.js# 素材资料库
├── chatbot_content.js  # 智能客服
├── biyuan_content.js   # 比元家办
└── assets/             # PDF等资源文件
```

### 版本号管理
- 每个 content 文件 URL 带有 `?v=YYYYMMDDXX` 参数
- 更新内容时需同步更新版本号以强制刷新浏览器缓存
- script_new.js 第17行管理所有 content 文件的版本引用

### CDN / 缓存策略
- GitHub Pages 默认缓存
- 通过版本号参数强制刷新：更改版本号即可

## 五、运营注意事项

1. **每周二/四**：小红书内容生成（素材资料库 → 3.2 小红书题材）
2. **每天10点前**：情报简报自动采集（飞书Bot推送）
3. **文章发布流程**：选题 → 4个Agent并行生产 → 飞书审核 → 公众号草稿箱
4. **违规风险点**：
   - 禁止返佣/回佣
   - 禁止内地异地签单
   - 禁止代签名
   - 禁止误导陈述
